  • isVisible

    A flag which mainly affects whether analytics events on the StorefrontSFMLView subclass are correctly triggered. This flag exists on the super class so its callback can be bundled into SFMLViewDelegate. It is the responsibility of the developer to set this flag appropriately. Changing this value is only functional if isLoaded is true (this is guarded internally so it is not necessary to check it manually). isVisible should be true when the view is on screen (viewDidAppear(_:) and viewDidLoad, when the app becomes active and it is the front-most view) and it should be false when the view leaves the screen (viewWillDisappear(_:) or the app goes into the background.

Accessibility Focus Tracking Properties

  • shouldTrackAccessibilityFocus

    Whether the view should track accessibility focus events. The value of this property mirrors isVisible and is updated automatically. You may also manually change this value if desired depending on the use case, however note that it will be overridden the next time isVisible changes. An example use case for updating it manually can be if you are displaying a custom view on top of SFMLView where isVisible wouldn’t be set to false but business logic dictates that accessibility elements on SFMLView should not interactable. In a case like this, you can manually set shouldTrackAccessibilityFocus to false before displaying the custom view so the last tracked element remains accurate. Upon dismissal of this temporary view, you may then call restoreFocusOnPreviouslyFocusedAccessibilityElement() to restore focus on the last accessibility element focused before presentation of the temporary view. If restoring the last focused accessibility element UX is not desired, simply never call restoreFocusOnPreviouslyFocusedAccessibilityElement().

    Declaration

    Swift

    @objc
public var shouldTrackAccessibilityFocus: Bool { get set }

Loading SFML

  • load(sfml:headerView:footerView:titlePreload:completion:)

    This method takes the given SFML Data and attempts to parse and render it.

    Note: if this method is called multiple times on the same instance, analytics events such as EV + open will fire again. The onus is on the client application to de-dupe as needed (this SDK makes no assumptions about the uniqueness of the content it is potentially re-loading).

    Parameters

    footerView
