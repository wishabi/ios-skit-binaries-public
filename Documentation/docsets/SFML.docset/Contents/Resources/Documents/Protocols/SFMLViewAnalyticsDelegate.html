SFMLViewAnalyticsDelegate

@objc
public protocol SFMLViewAnalyticsDelegate : NSObjectProtocol

Objects conforming to this protocol are notified of analytics events. After these methods are called, the object is free to send any relevant information to the desired analytics service.

  • sfmlViewEngagedVisitAnalyticsEvent(_:)

    Notifies the delegate that an engaged visit event was triggered. This event is only triggered once per “visible” session. When isVisible is to false, the engaged visit throttle is reset and the next time isVisible is true, and engaged visit event will trigger.

    Currently the following events trigger an engaged visit:

    • Waiting for <engagedVisitDelay> after load and not interacting with the view

    • Interacting with the view (scrolling, zooming, tapping/long pressing on an item
