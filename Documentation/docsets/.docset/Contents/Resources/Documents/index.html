SFML.framework

SFML.framework is a framework for rendering SFML. Its entry point is SFMLView, a UIView that takes SFML data and renders it in its own UIScrollView. See documentation for more details.

API Documentation

API documentation can be found at docs/index.html (open local file in your browser). This documentation is for external users of the framework and does not include any internal classes. See code comments for documentation on internal classes.

Documentation is generated using Jazzy.

Updating Documentation Steps

  1. Update documentation in code (must use the following formats for generation to work).

For multi-line method descriptions, use the following format:

   /**
   This is a multi-line method description.
   Use two asterisks at the start.

   - Parameter internalParameterName: Description of parameter.
   */
   func someMethod(with internalParameterName: String) {
     ...
   }

For single-line comments use triple forward slash:

   /// Flag for whether this object is special
   var isSpecial: Bool

To hide a public function from any documentation generation, use /// :nodoc:. For example, you may want to hide public implementations of protocols (e.g. UIScrollViewDelegate, etc.).

   /// :nodoc:
   func somePublicMethod() {
      ...
   }
  1. Install Jazzy and simply run jazzy from the SFML root project folder.

Accessibility

Every element that is accessible is represented by a UIAccessibilityElement (or subclass) object. This includes native elements (like UIViews, UIButtons, etc.). Their native accessibility properties are unused.

All accessiblity elements are then set as an array on the accessibilityElements property of the subview of the UIScrollView. This is to have more control over the order and behaviour of all accessibility elements. Heading accessibility elements are also created to make wayfinding accessible (via the Headings accessibility rotor option).
